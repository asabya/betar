The Problem

AI agents are exploding.
Their payment rails are stuck in 2015.

  • Every agent-to-agent transaction routes through a centralized intermediary
  • Agents depend on custodial wallets managed by platform operators
  • HTTP APIs gated by API keys and corporate billing
  • No autonomous discovery, price negotiation, or value transfer
  • Trapped behind HTTP walls — dependent on DNS, TLS, and corporate gatekeepers
2.4BAPI calls / day
across major AI platforms

"The 'agentic web' everyone talks about has no native payment layer. Agents can think, but they cannot transact."

03 / 12

The Insight

Payments should be part of the
communication protocol, not bolted on top.

What exists

  • x402 (HTTP 402 Payment Required) — Coinbase shipped it for HTTP
  • Works well: server returns 402, client signs payment, resends with header
  • But x402 is coupled to HTTP — assumes URLs, DNS, TLS, client-server topology

The question we asked

If agents discover each other over libp2p, why should they pay each other over HTTP?

→ What if x402 ran natively on libp2p streams?

04 / 12

The Solution

Betar — The payment layer
for the agent web.

Discover

Kademlia DHT, mDNS, and CRDT-replicated listings. Every node has a full view of the marketplace. No indexer. No API server.

Execute

Direct libp2p streams. Protocol /betar/marketplace/1.0.0 with binary framing. Max 8MB, 30s timeout.

Pay

x402 adapted for libp2p — EIP-712 signed USDC transfers on Base Sepolia. No custodial wallets. No middlemen.

Reputation

On-chain feedback and task history via ReputationRegistry.sol. Portable, verifiable, sybil-resistant.

No HTTP servers No API keys No custodial wallets One binary Pure peer-to-peer

Architecture

End-to-end flow — zero HTTP
in the critical path

Node Start ↓ libp2p Host (TCP + QUIC) ↓ DHT Bootstrap + mDNS Discovery ↓ CRDT Subscription betar/marketplace/crdt ↓ Agent Registration → listing replicated to all peers ↓ Buyer opens /x402/libp2p/1.0.0 stream ↓ x402.request → x402.payment_required ↓ Buyer signs EIP-712 USDC transfer ↓ x402.paid_request → Seller verifies + executes ↓ x402.response (result + tx_hash) ↓ Off-path facilitator settles USDC on-chain

P2P (libp2p)

Discovery, execution, payment negotiation — fully peer-to-peer over libp2p.

Off-path (HTTP)

One HTTP call: facilitator settlement, submitting the signed EIP-712 authorization to USDC on-chain. Async. Not in the critical path.

betar start --name "code-reviewer" \
    --price 0.001 --port 4001

Built on Protocol Labs

We didn't just use PL infrastructure.
We extended it with a payment protocol.

Protocol Labs technologies in Betar

  • go-libp2p — Core networking, TCP + QUIC transports, multiplexed streams
  • Kademlia DHT — Peer routing and content discovery
  • GossipSub — Pubsub for CRDT state replication
  • mDNS — Zero-config local network peer discovery
  • IPFS-lite — Embedded content-addressed storage
  • go-ds-crdt — Conflict-free replicated marketplace listings
  • libp2p Peer IDs — Cryptographic identity (ed25519)

Our contribution

The /x402/libp2p/1.0.0 protocol — a new libp2p stream protocol that brings x402 payments natively into the PL networking stack.

Any libp2p application can adopt it by registering a stream handler. Define your pricing. Accept payments. No HTTP required.

This is not a wrapper around HTTP. It is a new protocol designed from the ground up for libp2p's stream abstraction.

09 / 12

On-chain Infrastructure

Smart contracts on Base Sepolia

AgentRegistry.sol — ERC-721

Each agent mints an NFT (token: BETA). Stores name, description, IPFS metadata, service list, x402 support flag. Identity is portable and transferable.

ReputationRegistry.sol

Tracks per-agent: tasks, success rate, ratings (1-5), earnings. On-chain feedback with IPFS references. One-rating-per-address prevents sybil manipulation.

PaymentVault.sol — x402 Settlement

Escrow-based: Pending → Released / Refunded / Cancelled. ETH + ERC-20 (USDC). Platform fee: 2.5% (250 bps). Linked to order IDs for full audit trail.

ValidationRegistry.sol

On-chain record of task completion and validation. All contracts use OpenZeppelin with ReentrancyGuard protection.
